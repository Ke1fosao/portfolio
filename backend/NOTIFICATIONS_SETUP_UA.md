# Налаштування реальних сповіщень про заявки

## 1. Створіть локальний файл `.env`

У папці `backend` скопіюйте `.env.example` у `.env`. Файл `.env` ігнорується Git та не повинен потрапляти в публічний репозиторій або архів для клієнта.

## 2. Telegram

1. У `@BotFather` перевипустіть токен бота, оскільки попередній токен уже був надісланий у чат і більше не повинен вважатися секретним.
2. Вставте новий токен у `TELEGRAM_BOT_TOKEN`.
3. Напишіть боту будь-яке повідомлення.
4. Відкрийте в браузері `https://api.telegram.org/bot<NEW_TOKEN>/getUpdates`.
5. Скопіюйте значення `message.chat.id` у `TELEGRAM_CHAT_ID`.

## 3. Email через Gmail

1. Увімкніть двоетапну перевірку Google.
2. Створіть окремий пароль застосунку.
3. Вставте адресу в `EMAIL_HOST_USER`, `DEFAULT_FROM_EMAIL`, `ADMIN_NOTIFICATION_EMAIL`.
4. Вставте пароль застосунку в `EMAIL_HOST_PASSWORD`. Не використовуйте звичайний пароль від пошти.

## 4. Запуск

Після зміни `.env` перезапустіть backend. Виконайте міграції:

```bat
cd backend
venv\Scripts\activate
python manage.py migrate
python manage.py runserver
```

Заявка завжди спочатку зберігається в базі. Якщо Telegram або SMTP тимчасово недоступні, помилка записується в заявці, а з адмінпанелі можна повторити відправлення.
